<p>Register a callback for Scan Results. See <code dir="ltr" translate="no"><a href="/reference/android/net/wifi/WifiManager.ScanResultsCallback">ScanResultsCallback</a></code>.
 Caller will receive the event when scan results are available.
 Caller should use <code dir="ltr" translate="no"><a href="/reference/android/net/wifi/WifiManager#getScanResults()">WifiManager#getScanResults()</a></code> requires
 <code dir="ltr" translate="no"><a href="/reference/android/Manifest.permission#ACCESS_FINE_LOCATION">Manifest.permission.ACCESS_FINE_LOCATION</a></code> to get the scan results.
 Caller can remove a previously registered callback using
 <code dir="ltr" translate="no"><a href="/reference/android/net/wifi/WifiManager#unregisterScanResultsCallback(android.net.wifi.WifiManager.ScanResultsCallback)">WifiManager#unregisterScanResultsCallback(ScanResultsCallback)</a></code>
 Same caller can add multiple listeners.
 </p><p>
 Applications should have the
 <code dir="ltr" translate="no"><a href="/reference/android/Manifest.permission#ACCESS_WIFI_STATE">Manifest.permission.ACCESS_WIFI_STATE</a></code> permission. Callers
 without the permission will trigger a <code dir="ltr" translate="no"><a href="/reference/java/lang/SecurityException">SecurityException</a></code>.
 </p><p>

 <br/>
 Requires <code dir="ltr" translate="no"><a href="/reference/android/Manifest.permission#ACCESS_WIFI_STATE">Manifest.permission.ACCESS_WIFI_STATE</a></code></p>
