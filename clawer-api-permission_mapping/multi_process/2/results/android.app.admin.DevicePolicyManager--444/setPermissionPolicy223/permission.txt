<p>Set the default response for future runtime permission requests by applications. This
 function can be called by a device owner, profile owner, or by a delegate given the
 <code dir="ltr" translate="no"><a href="/reference/android/app/admin/DevicePolicyManager#DELEGATION_PERMISSION_GRANT">DELEGATION_PERMISSION_GRANT</a></code> scope via <code dir="ltr" translate="no"><a href="/reference/android/app/admin/DevicePolicyManager#setDelegatedScopes(android.content.ComponentName,%20java.lang.String,%20java.util.List&lt;java.lang.String&gt;)">setDelegatedScopes(ComponentName, String, List<string>)</string></a></code>.
 The policy can allow for normal operation which prompts the user to grant a permission, or
 can allow automatic granting or denying of runtime permission requests by an application.
 This also applies to new permissions declared by app updates. When a permission is denied or
 granted this way, the effect is equivalent to setting the permission * grant state via
 <code dir="ltr" translate="no"><a href="/reference/android/app/admin/DevicePolicyManager#setPermissionGrantState(android.content.ComponentName,%20java.lang.String,%20java.lang.String,%20int)">setPermissionGrantState(ComponentName, String, String, int)</a></code>.
 </p><p>
 As this policy only acts on runtime permission requests, it only applies to applications
 built with a <code dir="ltr" translate="no">targetSdkVersion</code> of <code dir="ltr" translate="no"><a href="/reference/android/os/Build.VERSION_CODES#M">Build.VERSION_CODES.M</a></code> or later.

 </p><p>
 NOTE: On devices running <code dir="ltr" translate="no"><a href="/reference/android/os/Build.VERSION_CODES#S">Build.VERSION_CODES.S</a></code> and above, an auto-grant
 policy will not apply to certain sensors-related permissions on some configurations.
 See <code dir="ltr" translate="no"><a href="/reference/android/app/admin/DevicePolicyManager#setPermissionGrantState(android.content.ComponentName,%20java.lang.String,%20java.lang.String,%20int)">setPermissionGrantState(android.content.ComponentName, java.lang.String, java.lang.String, int)</a></code> for the list of
 permissions affected, and the behavior change for managed profiles and fully-managed
 devices.</p>
