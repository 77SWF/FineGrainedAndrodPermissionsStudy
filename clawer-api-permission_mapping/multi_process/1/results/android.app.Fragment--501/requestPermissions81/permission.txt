<pre class="api-signature no-pretty-print" dir="ltr" translate="no">public final void requestPermissions (<a href="/reference/java/lang/String">String[]</a> permissions, 
                int requestCode)</pre><p>Requests permissions to be granted to this application. These permissions
 must be requested in your manifest, they should not be granted to your app,
 and they should have protection level <code dir="ltr" translate="no"><a href="/reference/android/content/pm/PermissionInfo">#PROTECTION_DANGEROUS dangerous</a></code>, regardless whether they are declared by
 the platform or a third-party app.
 </p><p>
 Normal permissions <code dir="ltr" translate="no"><a href="/reference/android/content/pm/PermissionInfo#PROTECTION_NORMAL">PermissionInfo.PROTECTION_NORMAL</a></code>
 are granted at install time if requested in the manifest. Signature permissions
 <code dir="ltr" translate="no"><a href="/reference/android/content/pm/PermissionInfo#PROTECTION_SIGNATURE">PermissionInfo.PROTECTION_SIGNATURE</a></code> are granted at
 install time if requested in the manifest and the signature of your app matches
 the signature of the app declaring the permissions.
 </p><p>
 Call <code dir="ltr" translate="no"><a href="/reference/android/app/Fragment#shouldShowRequestPermissionRationale(java.lang.String)">shouldShowRequestPermissionRationale(java.lang.String)</a></code> before calling this API
 to check if the system recommends to show a rationale UI before asking for a permission.
 </p><p>
 If your app does not have the requested permissions the user will be presented
 with UI for accepting them. After the user has accepted or rejected the
 requested permissions you will receive a callback on <code dir="ltr" translate="no"><a href="/reference/android/app/Fragment#onRequestPermissionsResult(int,%20java.lang.String[],%20int[])">onRequestPermissionsResult(int, java.lang.String[], int[])</a></code> reporting whether the
 permissions were granted or not.
 </p><p>
 Note that requesting a permission does not guarantee it will be granted and
 your app should be able to run without having this permission.
 </p><p>
 This method may start an activity allowing the user to choose which permissions
 to grant and which to reject. Hence, you should be prepared that your activity
 may be paused and resumed. Further, granting some permissions may require
 a restart of you application. In such a case, the system will recreate the
 activity stack before delivering the result to <code dir="ltr" translate="no"><a href="/reference/android/app/Fragment#onRequestPermissionsResult(int,%20java.lang.String[],%20int[])">onRequestPermissionsResult(int, java.lang.String[], int[])</a></code>.
 </p><p>
 When checking whether you have a permission you should use <code dir="ltr" translate="no"><a href="/reference/android/content/Context#checkSelfPermission(java.lang.String)">Context.checkSelfPermission(String)</a></code>.
 </p><p>
 Calling this API for permissions already granted to your app would show UI
 to the user to decide whether the app can still hold these permissions. This
 can be useful if the way your app uses data guarded by the permissions
 changes significantly.
 </p><p>
 You cannot request a permission if your activity sets <code dir="ltr" translate="no"><a href="/reference/android/R.styleable#AndroidManifestActivity_noHistory">noHistory</a></code> to
 <code dir="ltr" translate="no">true</code> because in this case the activity would not receive
 result callbacks including <code dir="ltr" translate="no"><a href="/reference/android/app/Fragment#onRequestPermissionsResult(int,%20java.lang.String[],%20int[])">onRequestPermissionsResult(int, java.lang.String[], int[])</a></code>.
