<p>Returns the set of permissions corresponding to a given <code dir="ltr" translate="no">String</code>
 representation.

 </p><p> The <code dir="ltr" translate="no">perms</code> parameter is a <code dir="ltr" translate="no">String</code> representing the
 permissions. It has 9 characters that are interpreted as three sets of
 three. The first set refers to the owner's permissions; the next to the
 group permissions and the last to others. Within each set, the first
 character is <code dir="ltr" translate="no">'r'</code> to indicate permission to read, the second
 character is <code dir="ltr" translate="no">'w'</code> to indicate permission to write, and the third
 character is <code dir="ltr" translate="no">'x'</code> for execute permission. Where a permission is
 not set then the corresponding character is set to <code dir="ltr" translate="no">'-'</code>.

 </p><p> <b>Usage Example:</b>
 Suppose we require the set of permissions that indicate the owner has read,
 write, and execute permissions, the group has read and execute permissions
 and others have none.
