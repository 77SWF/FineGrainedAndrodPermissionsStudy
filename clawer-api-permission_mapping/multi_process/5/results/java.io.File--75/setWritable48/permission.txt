<p>Sets the owner's or everybody's write permission for this abstract
 pathname.

 </p><p> The <code dir="ltr" translate="no"><a href="/reference/java/nio/file/Files">Files</a></code> class defines methods that operate on
 file attributes including file permissions. This may be used when finer
 manipulation of file permissions is required.</p><table class="responsive">
    <tbody><tr><th colspan="2">Parameters</th></tr>
      <tr>
        <td><code dir="ltr" translate="no">writable</code></td>
        <td width="100%">
          <code dir="ltr" translate="no">boolean</code>: If <code dir="ltr" translate="no">true</code>, sets the access permission to allow write
          operations; if <code dir="ltr" translate="no">false</code> to disallow write operations<p></p></td>
      </tr>
      <tr>
        <td><code dir="ltr" translate="no">ownerOnly</code></td>
        <td width="100%">
          <code dir="ltr" translate="no">boolean</code>: If <code dir="ltr" translate="no">true</code>, the write permission applies only to the
          owner's write permission; otherwise, it applies to everybody.  If
          the underlying file system can not distinguish the owner's write
          permission from that of others, then the permission will apply to
          everybody, regardless of this value.<p></p></td>
      </tr>
    </tbody></table><table class="responsive">
      <tbody><tr><th colspan="2">Returns</th></tr>
      <tr>
        <td><code dir="ltr" translate="no">boolean</code></td>
        <td width="100%"><code dir="ltr" translate="no">true</code> if and only if the operation succeeded. The
          operation will fail if the user does not have permission to change
          the access permissions of this abstract pathname.<p></p></td>
      </tr>
    </tbody></table>
